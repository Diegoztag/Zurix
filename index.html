Zurix

Diseño a velocidad luz con zurix, framework HTML, mobile first y flexbox, agiliza tus proyectos web!!

zurix.zip

Zurix es un framework exclusivamente CSS para crear diseños web responsive (RWD) y mobile first.

Desarolla geniales proyectos para cualquier tamaño de pantalla fácilmente, con zurix es alucinante lo fácil que resulta pues es senmántico, legible, flexible y absolutamente personalizable.

Creado en SASS, destinado para maquetación o estructura pura y en tiempo récord, no añade estilos predeterminados para evitar conflictos con proyectos en desarrollo y así olvidarse de las colisiones de estilos css framework-diseñador, dotado con todo el poder de flexbox!! es hora de olvidar los floats.

First mobile

Zurix esta diseñado para ser first mobile, usted puede iniciar en dispositivos pequeños primero y a medida que el navegador se hace mas grande se puede ir añadiendo mas complejidad para un diseño organizado,aunque no esta escrito en piedra, también puede iniciar en dispositivos grandes, solo es cuestión de gustos.

Versatilidad

Destinado para la maquetación o estructura pura, zurix no tiene css visual preestablecido para que esto no choque con el css del diseñador, cosa que hace de este framework genial para incluir en proyectos en desarrollo con la seguridad de que no habrá inconvenientes o rupturas del esquema

Intuitivo

Esta diseñado para ser rápido, sencillo y generar una curva de aprendizaje mínima, los nombres de las clases son muy fáciles de memorizar, utiliza zurix por un rato y en un pestañeo lo dominaras como un pro.

Personalizable

En su versión Sass zurix es increíblemente personalizable, solo basta con modificar los valores de variables declaradas por defecto o crear propias, ademas que es muy liviano a comparación de otros frameworks,

Empezando a utilizar zurix

Contenedores padre

Una familia es un gran ejemplo para aprender y es la base de todo, en Zurix se manejan los contenedores con el nombre de papas (.papa) e hijos (.hijo) para tener una mejor comprensión y organización de contenedores, se aconseja utilizar la etiqueta div para generar los contenedores, veras que es muy fácil memorizar las clases de este framework y rápidamente estarás haciendo estructuras HTML con estilo y a la velocidad de la luz.

Hay varios puntos que se tienen que dejar en claro en cuanto a el uso de contenedores papas e hijos en zurix:

  • Los papas con los contenedores principales y los hijos son divisiones del contenedor papa.
  • Todo contenedor papa debe tener al menos un hijo y los hijos deben estar dentro de un papa.
  • Los contenedores hijos llevan la clase .hijo y ademas otra clase que describa su anchura con la clase ".tamaño-x" o si es una fracción ".tamaño-x-y".

Ancho de los contenedores .papa

Como en todas las familias los padres son los eslabones primordiales de la casa, al igual en zurix los grupos con la clase .papa son los contenedores que albergan los divs .hijos para su organización y función. Los contenedores .papa son los estelares del framework zurix y contienen las siguientes propiedades

					
						.papa {
								position: relative;
								max-width: 1100px;
								margin: 0 auto;
						}

Como las propiedades mismas lo indican, el ancho maximo de los contenedores papas indican hasta cual es el limite que llegaran en este caso 1100px, con el margin 0 auto nos aseguramos que el contenedor estara centrado con respecto al cuerpo de la pagina o algún contenedor designado como principal

Si tu ventana o viewport tiene una dimensión mayor a la designada en este caso 1100px, el contenedor quedara en el centro y los espacios que sobran se repartirán por igual hacia los lados, si por algún motivo no quisieras que el ancho máximo del contenedor fuera de 1100px simplemente basta con sobrescribir en un css el valor nuevo que se le va a asignar, o mejor aun si usas sass en el archivo "zurix.scss" puedes modificar su valor directamente en su variable nombrada $max-width.

Media Queries

Entrando en materia cuando hablamos de media queries, breakpoints o puntos de quiebre nos referimos a el punto en el que el tamaño del viewport o pantalla presenta cambios para adaptarse al tipo de resolución o dimensión del viewport y así evitar que las partes del diseño no se rompan y se genere un caos total.

Para que tu desarollo o proyecto web identifique los cambios de los media queries es necesario declarar una etiqueta meta llamada "viewport". La etiqueta viewport nos permite a los que construimos sitios web o web apps, definir el ancho, alto y escala del área usada por el navegador para mostrar contenido. A continuación la configuración más común de una etiqueta meta, esta etiqueta se debe de incluir en el html entre los tags o etiquetas "head".

					
						<meta name = "viewport" content = "width=device-width, initial-scale=1, maximum-scale=1">

Como en todos o la mayoria de framework frontend css ofrecen la bondad de manipular media queries para tu beneficio, pues zurix no es la excepción

Zurix presenta por default varios breakpoints:

  • Base: 320px en adelante.
  • Movil: 480px en adelante.
  • Tablet: 768px en adelante.
  • Web: 1024px en adelante.
  • Hd: 1500px en adelante.

Hay que recordar que zurix es un framework orientado a mobile first entonces los estilos que se determinen para un tamaño se heredan para los breakpoints sucesores.

En la versión Sass de zurix en el archivo base nombrado "zurix.scss" se pueden editar los tamaños de medias queries que se ofrecen por default declarados en un map estos definen los tamaños de los contenedores a través de clases, es por eso que se recomienda utilizar zurix en su versión sass si desea tener mayor personalización de framework.
