Zurix

Diseño a velocidad luz con zurix, framework HTML, mobile first y flexbox, agiliza tus proyectos web!!

zurix.zip

Zurix es un framework exclusivamente CSS para crear diseños web responsive (RWD) y mobile first.

Desarolla geniales proyectos para cualquier tamaño de pantalla fácilmente, con zurix es alucinante lo fácil que resulta pues es senmántico, legible, flexible y absolutamente personalizable.

Creado en SASS, destinado para maquetación o estructura pura y en tiempo récord, no añade estilos predeterminados para evitar conflictos con proyectos en desarrollo y así olvidarse de las colisiones de estilos css framework-diseñador, dotado con todo el poder de flexbox!! es hora de olvidar los floats.

Mobile first

Zurix esta diseñado para ser mobile first, usted puede iniciar en dispositivos pequeños primero y a medida que el navegador se hace mas grande se puede ir añadiendo mas complejidad para un diseño organizado,aunque esto no esta escrito en piedra, también puedes iniciar en dispositivos grandes, solo es cuestión de gustos.

Versatilidad

Destinado para la maquetación o estructura pura, zurix no tiene css visual preestablecido para que esto no choque con el css del diseñador, cosa que hace de este framework genial para incluir en proyectos en desarrollo con la seguridad de que no habrá inconvenientes o rupturas en esquema visual.

Intuitivo

Esta diseñado para ser rápido, sencillo y generar una curva de aprendizaje mínima, los nombres de las clases son muy fáciles de memorizar, utiliza zurix por un rato y en un pestañeo estarás haciendo magia.

Personalizable

En su versión Sass zurix es increíblemente personalizable, solo basta con modificar los valores de variables declaradas por defecto o crear tus propias variables, ademas que es muy liviano a comparación de otros frameworks,

Iniciando con zurix

Contenedores padre

Una familia es un gran ejemplo para aprender y es la base de todo, en Zurix se manejan los contenedores con el nombre de papas (papa) e hijos (hijo) para tener una mejor comprensión y organización de contenedores, aquí se aconseja utilizar la etiqueta div para generar los contenedores, veras que es muy fácil memorizar las clases de este framework y rápidamente estarás haciendo estructuras HTML con estilo y a la velocidad de la luz.

Hay varios puntos que se tienen que dejar en claro en cuanto a el uso de contenedores papas e hijos en zurix:

  • Los papas con los contenedores principales y los hijos son divisiones del contenedor papa.
  • Todo contenedor papa debe tener al menos un hijo y los hijos deben estar dentro de un papa.
  • Los contenedores hijos llevan la clase .hijo y ademas otra clase que describa su anchura con la clase "tamaño-x" o si es una fracción "tamaño-x-y".

Ancho de los contenedores padres

Como en todas las familias los padres son los eslabones primordiales de la casa, al igual en zurix los grupos con la clase .papa son los contenedores que albergan los divs .hijos para su organización y función. Los contenedores .papa son los estelares del framework zurix y contienen las siguientes propiedades

					
						.papa {
								position: relative;
								max-width: 1100px;
								margin: 0 auto;
						}

Como las propiedades mismas lo indican, el ancho maximo de los contenedores papas indican hasta cual es el limite que llegaran en este caso 1100px, con el margin 0 auto nos aseguramos que el contenedor estara centrado con respecto al cuerpo de la pagina o algún contenedor designado como principal

Si tu ventana o viewport tiene una dimensión mayor a la designada en este caso 1100px, el contenedor quedara en el centro y los espacios que sobran se repartirán por igual hacia los lados, si por algún motivo no quisieras que el ancho máximo del contenedor fuera de 1100px simplemente basta con sobrescribir en un css el valor nuevo que se le va a asignar, o mejor aun si usas sass en el archivo "zurix.scss" puedes modificar su valor directamente en su variable nombrada $max-width.

Media queries

Entrando en materia cuando hablamos de media queries, breakpoints o puntos de quiebre nos referimos a el punto en el que el tamaño del viewport o pantalla presenta cambios para adaptarse al tipo de resolución o dimensión del viewport y así evitar que las partes del diseño no se rompan y se genere un caos total.

Para que tu desarollo o proyecto web identifique los cambios de los media queries es necesario declarar una etiqueta meta llamada "viewport". La etiqueta viewport nos permite a los que construimos sitios web o web apps, definir el ancho, alto y escala del área usada por el navegador para mostrar contenido. A continuación la configuración más común de una etiqueta meta, esta etiqueta se debe de incluir en el html entre los tags o etiquetas "head".

					
						<meta name = "viewport" content = "width=device-width, initial-scale=1, maximum-scale=1">

Como en todos o la mayoria de framework frontend css ofrecen la bondad de manipular media queries para tu beneficio, pues zurix no es la excepción

Zurix presenta por default varios breakpoints o tamaños:

  • Base: 320px en adelante.
  • Movil: 480px en adelante.
  • Tablet: 768px en adelante.
  • Web: 1024px en adelante.
  • Hd: 1500px en adelante.

Hay que recordar que zurix es un framework orientado a mobile first entonces los estilos que se determinen para un tamaño se heredan para los breakpoints sucesores.

En la versión Sass de zurix en el archivo base nombrado "zurix.scss" se pueden editar los tamaños de medias queries que se ofrecen por default declarados en un map estos definen los tamaños de los contenedores a través de clases, es por eso que se recomienda utilizar zurix en su versión sass si desea tener mayor personalización de framework.

					
						$breakpoints:(
								base:  			320px,
								movil: 			480px,
								tablet:			768px,
								web:   			1024px,
								hd:    			1400px
						)
Ancho de elementos del DOM

Con zurix es fácil asignar y controlar el manejo de los anchos de elementos del HTML, como por ejemplo párrafos, imágenes, vídeos, contenedores, menús, todo lo que se te ocurra, esto hace a zurix una herramienta extremadamente fácil de utilizar para controlar rápidamente el width o anchura de objetos del html.

Para realizar esto simplemente es necesario asignar una clase a la etiqueta del elemento que se quiere manipular, esta clase estará designada por las medidas de los breakpoints (base,movil,tablet,web,hd) y el numero entero del (5 al 100 pero siempre siendo múltiplo de 5) que representa el porcentaje que se asignara de anchura al elemento.

Ahora bien la clase quedaría dada por la siguiente expresión: "medida-x" siendo "medida" como mencionamos anteriormente el tamaño de un breakpoint y "x" el porcentaje de anchura que tendrá el elemento. Veamos unos ejemplos:

Si queremos un imagen que tenga un 50% del ancho del navegador en su tamaño web "1024px en adelante" entonces añadiremos a la etiqueta "img" la siguiente clase:

					
						<img class = "web-50" src="http://lorempixel.com/100/100/">

Muy bien!! cuando el viewport sobrepasa los 1024px se activa nuestra clase y se adapta el ancho del elemento, pero que pasa cuando la ventana es menor de 1024px??, simple, la clase con el ancho que declaramos ya no tiene efecto y el ancho de la imagen se devuelve a su estado original por que la ventana ya no es de tamaño "web", si no se redujo a tamaño tableta o menor.

Hagamos otro ejemplo para dejar bien claro esto que mencione, veamos en esta ocasión quiero la misma imagen, pero quiero que en su ancho para móviles tenga un 50%, cuando salte al tamaño para tableta tenga digamos un 80% y al pasar al tamaño web un 30% de anchura, entonces la etiqueta quedaría declarada con estas clases:

					
						<img class = "movil-50 tablet-80 web-30" src="http://lorempixel.com/100/100/">

A que te recuerda esto?? si alguna vez usaste boostraps o foundation utilizabas las medidas para indicar hasta donde se mantendrían los anchos de las clases, por ejemplo bootstraps utiliza las medidas de sus clases (xs,sm,md,ls,xl), si te pones a pensar es realmente casi lo mismo en zurix solo que aquí se declaran (base,movil,tablet,web,hd)

Hay que puntualizar una cosa, he de recordar que las medidas que mencione antes (base,movil,etc) son las de default y son las medidas mas utilizadas en cuanto a la dimensión de pantallas como por ejemplo en celulares, en tablets, pantallas de computadoras o laptops, pero si utilizas Sass puedes fácilmente crear mas breakpoints personalizados e incluir sus medidas a esta función.
